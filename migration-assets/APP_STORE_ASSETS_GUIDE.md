## Store Listing Content

### Text Field Limits

| Field | iOS Limit | Android Limit | Tips |
|-------|-----------|---------------|------|
| App Name | 30 chars | 30 chars | Include key feature keyword |
| Subtitle | 30 chars | N/A | Secondary benefit/feature |
| Short Description | N/A | 80 chars | Hook with main value prop |
| Full Description | 4000 chars | 4000 chars | Features, benefits, social proof |
| Keywords | 100 chars | N/A | Comma-separated, no spaces after commas |
| Promotional Text | 170 chars | N/A | Updatable without review |
| What's New | 4000 chars | 500 chars | Version changes, new features |

## Screenshot Capture Guide

### Using Expo/React Native

```bash
# Capture iOS Simulator Screenshots
xcrun simctl io booted screenshot ~/Desktop/screenshot.png

# Capture Android Emulator Screenshots
adb exec-out screencap -p > ~/Desktop/screenshot.png
```

### Automated Screenshot Capture

```typescript
// Using expo-screen-capture for in-app screenshots
import * as ScreenCapture from 'expo-screen-capture';

const captureScreen = async () => {
  const uri = await ScreenCapture.captureScreenAsync();
  console.log('Screenshot saved:', uri);
};
```

### Screenshot Best Practices

1. **Clean Status Bar**
   - Full battery (100%)
   - Full signal bars
   - Time set to 9:41 AM (Apple tradition)
   - No notifications

2. **Realistic Data**
   - Use realistic but aspirational numbers
   - Show meaningful content, not lorem ipsum
   - Feature diverse artist names and content

3. **Consistent Styling**
   - Same device frame across all screenshots
   - Consistent caption styling
   - Unified background treatment

4. **Localization Ready**
   - Leave space for text expansion (30% for non-Latin)
   - Use separate layers for text
   - Consider right-to-left layouts
